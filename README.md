# 트랜잭션

<details>
<summary>트랜잭션이란 무엇인가요? 4가지 원칙을 포함해서 설명해 주세요.</summary>

트랜잭션은 데이터베이스의 상태를 변화시키기 위한 작업 수행의 논리적 단위를 
말합니다

여기서 등장하는 작업 단위는 사용자가 특정 기능의 수행을 위해 SQL 작업을 
묶은 단위를 말할수있습니다

쉽게 설명하면 
A가 B에게 돈을 송금하고 B가 안전하게 입금을 받았을때
한 작업단위가 완전히 실행되었다 말할수 있습니다.
만약 A는 보냈지만 시스템 내부 문제로 B가 못받으면?
해당 작업은 완료되지않는다


자연스럽게 트랜잭션의 특성 4가지 중 하나인 원자성을 지킨 상황이된다

4가지 특성 중 원자성은 트랜잭션 내부에서 실행된 작업이 완료되거나,
문제가 생겨 rollback 하거나 둘 중 하나만 수행된다

일 부분만 성공이 될순없다.

일관성은 모든 트랜잭션은 일관성이 있는 데이터베이스 상태를 유지해야하는 
규칙이 존재하는데 이걸 ‘무결성 제약 조건’이라 말하며
은행에서 완전히 똑같은 적금을 다른 2명이 만들었는데, A는 이자가 4% B는 6% 
이렇게 제각각 다를수 없다는것이다.

정해진 데이터 타입이나 혹은 값이 있으면 반드시 모든 트랜잭션은 똑같이 
지켜져야한다

격리성은 각각의 트랜잭션이 서로 간섭없이 독립적으로 수행되어야하는데, 
이유는 트랜잭션은 기본적으로 원자성, 일관성 지속성을 보장해주는데 격리성의 
경우 완전히 보장하려면 각각의 트랜잭션을 순서대로 처리해야한다
이는 동시성 처리 성능을 매우 저하시키므로 격리성의 수준을 여러 단계로 
분리해놓았다

지속성
성공적으로 수행된 트랜잭션은 영구적으로 기록되게 된다
이유는 중간에 시스템에 문제가 발생했을때 데이터베이스 로그를 참고해서 
성공했던 트랜잭션을 복구할수 있다.

<details>
<summary> Mock이 무엇인지 설명해 주세요. </summary>

Mock 이란

실제 객체를 만들기엔 비용과 시간이 많이 들거나 의존성이 길게 걸쳐져있어 제대로 구현하기가 어려울 경우, 가짜 객체를 만들어 사용하는데
이것을 Mock이라고 합니다

즉, 실제와 동일한 기능은 하진 않지만 대략 호출을 하면, 이런 결과 이런 기능을 할것이라고 알려주는 용도로 사용한다

목 객체가 필요한 경우는
테스트 작성을 위한 환경 구축이 어려울때
테스트가 특정 경우나 순간에 의존적인 경우
혹은 시간이 오래걸리는 경우에 필요하며

주의할점이 있는데
Mock을 사용할 경우 테스트 케이스 유지에 복잡성이 더해지기 때문에,
의존성이 적은 구조로 프로그래밍 해야하며
실제 객체로 작동을 해봤을때 예상한것처럼 정상 작동이 하지 않을 수도 있다 이유는 Mock은 실제 객체가 아닌 흉내만 내는 가짜 객체이기 떄문이다

Mock을 무분별하게 사용할경우 리팩토링에서 에러가 발생해
테스트에 대한 유지보수 비용이 크게 발생할 수 있다.

객체에 필요한 행동만 주의 깊게 만들어 Mock 객체를 형성하는 것이 바람직하다 불필요하면 상태 검증 테스트가 가능하다면 만들지 않는 것을 권장합니다

</details>
